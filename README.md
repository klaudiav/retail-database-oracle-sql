# retail-database-oracle-sql
Retail Database Design & Implementation (Oracle SQL)

Overview

This project consists of designing and implementing a simple relational database for a retail business, supporting operations such as product management, customer records, and order tracking.

Objectives
- Design a structured database for retail operations
- Model relationships between key business entities
- Ensure data integrity and consistency
- Simulate real-world business processes

Tools & Technologies
- Oracle SQL
- Oracle SQL Developer

Database Structure
The database includes the following main entities:
- Products
- Suppliers
- Customers
- Orders
- Order Details

Key Features
- Implementation of primary and foreign keys
- Definition of relationships (1:N, N:N)
- Data normalization for efficient storage
- Use of constraints (PRIMARY KEY, FOREIGN KEY, UNIQUE)

Operations Implemented
- Table creation (DDL)
- Data insertion (INSERT statements)
- Data updates and modifications
- Schema changes (ALTER TABLE)
- Data management and validation

Use Case
The database supports:
- Inventory tracking
- Order management
- Customer data handling
- Business reporting

Files
- SQL scripts (table creation, inserts, updates) by parts + complete
